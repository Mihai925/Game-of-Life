Design Patterns


Because I have given quite a significant attention to design patterns, I have chosen to talk more about them instead of implementing an extension. Throughout the code, I have used three design patterns: Adapter Pattern, MVC Pattern, Factory Pattern.

 

Adapter Pattern

The adapter pattern is generally used to translate a certain interaface into another compatible one.
In this specific case, I have found that the endgeless grid could be a way to translate certain instructions to the normal grid. The GridAdapter class takes a normal grid as a parameter:
GridAdapter g = new GridAdapter(new Grid());

The main function used throught the GridAdapter class is the one that translates the coordinates from overflowing points into ones within the valid range of the grid:
private int getLC(int x) { return (x+Grid.SIZE) % Grid.SIZE; }

 

MVC Pattern

The MVC Pattern is probably the most predominant design pattern from this script. The way I designed it to function is the following:

  1. The cell action listener (Controller) notices the user's input and transmits the new state of the cell to the current Life object.
  2. The Life object takes the information and requests modifications in the grid (Model)
  3. The GridAdapter passes the coordinates through its filters and furthers them to the Grid
  4. The Grid (Model) makes the modifications and requests changes to the GUI
  5. The GUI (View) changes and the user can see the modifications

Factory Pattern

I have implemented an easily modified version of the Factory Pattern to give a certain degree of readability and agility to the code. Because the GUI is created after the first grid, I enabled the grid to be created without having any specific gui when the program starts. After each step, however, a new grid needs to be created. This time, however, it was crucial that the grid would be initialized with the gui as a parameter so that the MVC pattern will function well.
In order to enable all these, I have created a separate factory class which contains two static methods (each returning a certain type of grid).
